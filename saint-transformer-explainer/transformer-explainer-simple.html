What is SAINT?

SAINT stands for Self-Attention Supervised Tabular Transformers. It's a neural network architecture designed specifically for tabular data — data organized in tables, like spreadsheets.

In finance, we work with tabular data all the time: companies in rows, financial metrics in columns. SAINT is particularly good at this because it can learn relationships between different metrics — not just look at each one in isolation.

Why "Self-Attention"?
Self-attention means the model can "pay attention" to different parts of its input simultaneously. For a company with 100+ financial metrics, the model learns which metrics matter most for prediction, and how they interact with each other.

The Key Insight

Traditional models might treat "price-to-earnings ratio" as a standalone signal. SAINT learns that "P/E ratio" means something different when "momentum" is high versus when it's low. It captures these contextual relationships automatically.

Organizing Financial Information

Before SAINT can learn anything, we need to organize our data. Each row represents one company at one point in time. Each column is a financial metric — what we call features.

Value Metrics
P/E ratio, price-to-book, dividend yield — is the stock cheap or expensive?
📈
Growth Metrics

⭐
Quality Metrics
Profitability, return on equity — is the company well-run?
🚀
Momentum Metrics
Price momentum, volatility — how is the stock moving?

Each feature gets converted into a numeric representation — a list of 256 numbers. This is called an embedding. Think of it as translating the raw number "15.2" into a richer description that captures its meaning.

What's an Embedding?
An embedding is a way of representing information as numbers. Instead of just storing "P/E = 15.2", we store 256 numbers that encode what "15.2" means in context — is it high or low? What range is typical? What does it predict? The model learns these representations automatically.
03 / 05

How SAINT Learns Relationships

This is the core of the SAINT architecture. After converting features into embeddings, the model runs them through 6 layers of self-attention.

Each layer does something simple but powerful: it lets every feature "ask questions" of every other feature. "Price-to-earnings" might ask "momentum": "Does my value mean something different when stocks are rising fast?"

1

Feature Embeddings Enter

Each feature (P/E, momentum, etc.) is now a 256-dimensional vector

2

Self-Attention Layer

Each feature computes its relationship to all other features

3

Feed-Forward Processing

Each feature's representation is refined based on what it learned

4

Repeat 6 Times

Going through multiple layers lets the model learn increasingly complex relationships

5

CLS Token Aggregates

A special token collects information from all features into a single summary

Why Multiple Layers?
One layer can learn simple relationships: "high momentum + low volatility" is good. Six layers deep, the model can learn complex combinations: "high momentum + low volatility + cheap valuation + strong profitability" might be even better, but only in certain market conditions.

04 / 05

How the Model Learns

SAINT uses self-supervised learning — a fancy way of saying it learns from the data itself, without explicit labels. It does this through two training tasks.

🔒
Task 1: Masked Reconstruction
Hide some features and ask the model to guess them. This forces it to learn relationships — you can't guess a hidden value if you don't understand how it relates to the visible ones.
⏭️
Task 2: Temporal Prediction
Show this month's data and ask the model to predict next month's features. This teaches it about dynamics — how indicators evolve over time.

Both tasks run for 10 epochs each. An epoch means going through the entire dataset once, with the data shuffled randomly each time to help the model generalize.

Why Self-Supervised?
We don't need to manually label data as "this is a good stock" or "this is a bad stock." Instead, we create learning tasks from the data itself. By learning to reconstruct masked features and predict future values, the model builds rich representations that happen to be useful for predicting returns.
05 / 05

From Model to Portfolio

After training, we use the SAINT model to generate predictions. But we don't just take the raw model output — we combine it with other techniques in a two-stage approach.

1

Extract CLS Embeddings

After SAINT processes all features, the CLS token contains a compressed representation of everything the model learned about that company at that time

2

Dimensionality Reduction

Use PCA to reduce 256 dimensions to 100, keeping the most important information

3

Random Forest Ensemble

Train 200 decision trees on the embeddings to predict actual returns

4

KNN Smoothing

Apply k-nearest neighbors to smooth predictions, reducing noise

5

Build Portfolio

Convert predictions to positions with risk controls (64+ positions, balanced long/short)

Why Two Stages?

SAINT builds rich representations of companies through self-supervised learning. But for the final prediction, we use a Random Forest — it's robust, handles noise well, and provides natural uncertainty quantification. The KNN smoothing adds a final layer of stability. This combination performs better than using either approach alone.

Risk Management
The model generates predictions, but we don't follow them blindly. We cap maximum position sizes, limit sector exposure, and maintain diversification. The AI is a tool that informs decisions — human judgment and risk controls remain essential.

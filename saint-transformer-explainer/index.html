01 / 06 Overview

The Big Picture

The model combines deep learning with quantitative finance. A SAINT (Self-Attention Supervised Tabular Transformer) encoder with 6 layers learns rich representations from financial features through self-supervised tasks.

📊

Rich Data

Monthly indicators across multiple families: technical, low-risk, growth, quality, value, size, and liquidity.

🧠

Neural Architecture

6-layer transformer encoder with multi-head self-attention (8 heads) learns contextual relationships.

🎯

Self-Supervised

Pretraining with masked reconstruction and temporal prediction teaches feature relationships.

📈

Return Prediction

Learned embeddings feed a Random Forest + KNN ensemble to predict future returns.

02 / 06 Input Data

The Feature Matrix

We organize financial indicators into a structured matrix for neural processing. Each row represents a company in a given month; each column is a financial feature.

X ∈ ℝN×m

N = companies/months  ·  m = features

Continuous features are standardized and transformed by a linear layer with ReLU. Discrete features use learned embeddings. A special CLS token is added as an aggregate sequence representation.

Embedding dimension: d = 256  ·  Batch size: B = 128
Features include Value, Growth, Quality, Technical indicators, etc.

05 / 06 Prediction

From Embeddings to Returns

After pretraining, we extract the CLS token from each observation as a compact embedding. These 256-dimensional vectors capture all feature information in a dense representation.

ecompany,month = CLSfinal ∈ ℝ256

Embeddings are standardized, reduced via PCA (256→100), and used as features in a Random Forest ensemble (200 trees) with KNN smoothing (k=10).

Learning rate: 1e-4  ·  Epochs: 10 per task
PCA retains 100 components  ·  RF uses 200 estimators

Modelo de Ações Baseado em Transformer

Uma abordagem de deep learning para prever retornos de ações usando transformers auto-supervisionados e indicadores financeiros. Self-attention, embeddings e aprendizado de representação aplicados aos mercados.

01 / 06 Visão Geral

O Grande Quadro

O modelo combina aprendizado profundo com finanças quantitativas. Um encoder SAINT (Self-Attention Supervised Tabular Transformer) com 6 camadas aprende representações ricas de features financeiras através de tarefas auto-supervisionadas.

📊

Dados Ricos

Indicadores mensais de múltiplas famílias: técnicos, baixo risco, crescimento, qualidade, valor, tamanho e liquidez.

🧠

Arquitetura Neural

Encoder transformer com 6 camadas de multi-head self-attention (8 cabeças) aprende relações contextuais entre features.

🎯

Auto-Supervisionado

Pré-treinamento com reconstrução mascarada e predição temporal ensina o modelo a entender relações entre features.

📈

Predição de Retornos

Embeddings aprendidos alimentam um ensemble de Random Forest + KNN para prever retornos futuros.

02 / 06 Dados de Entrada

A Matriz de Features

Organizamos indicadores financeiros em uma matriz estruturada para processamento neural. Cada linha representa uma empresa em um determinado mês; cada coluna é um feature financeiro.

X ∈ ℝN×m

N = empresas/meses  ·  m = features

Features contínuas são padronizadas e transformadas por uma camada linear com ReLU. Features discretas usam embeddings aprendidos. Um token CLS especial é adicionado como representação agregada.

Dimensão do embedding: d = 256  ·  Tamanho do batch: B = 128
Features incluem indicadores de Value, Growth, Quality, Technical, etc.

04 / 06 Treinamento

Pré-Treinamento Auto-Supervisionado

Duas tarefas de pretexto ensinam o modelo a entender relações entre features sem rótulos explícitos. O modelo aprende a reconstruir features mascaradas e prever valores futuros.

Tarefa 1

20% das features observadas são aleatoriamente mascaradas. O modelo deve prever seus valores originais, forçando-o a aprender relações entre diferentes indicadores financeiros.

05 / 06 Predição

De Embeddings a Retornos

Após o pré-treinamento, extraímos o token CLS de cada observação como embedding compacto. Esses vetores de 256 dimensões capturam toda a informação das features em uma representação densa.

eempresa,mês = CLSfinal ∈ ℝ256

Os embeddings são padronizados, reduzidos via PCA (256→100), e usados como features em um ensemble de Random Forest (200 árvores) com suavização KNN (k=10).

Learning rate: 1e-4  ·  Épocas: 10 por tarefa
PCA retém 100 componentes  ·  RF usa 200 estimadores

Construção da Carteira

Predições de retorno são convertidas em sinais de trading através de limiares, depois transformadas em pesos de carteira respeitando restrições de exposição.

Si = +1 se r̂i > τL
Si = −1 se r̂i < τS
Si = 0 caso contrário
τL = limiar long  ·  τS = limiar short
Exposição máxima long: 130%  ·  Exposição máxima short: 30%

01 / 05

O que é SAINT?

SAINT significa Self-Attention Supervised Tabular Transformers (Transformadores Auto-Supervisionados para Dados Tabulares). É uma arquitetura de rede neural projetada especificamente para dados tabulares — dados organizados em tabelas, como planilhas.

Em finanças, trabalhamos com dados tabulares o tempo todo: empresas nas linhas, métricas financeiras nas colunas. SAINT é particularmente bom nisso porque pode aprender relacionamentos entre diferentes métricas — não apenas olhar para cada uma de forma isolada.

Por que "Self-Attention"?
Self-attention significa que o modelo pode "prestar atenção" a diferentes partes de sua entrada simultaneamente. Para uma empresa com 100+ métricas financeiras, o modelo aprende quais métricas importam mais para previsão, e como elas interagem entre si.

O Insight Principal

Modelos tradicionais podem tratar "índice preço/lucro" como um sinal isolado. SAINT aprende que "P/L" significa algo diferente quando "momentum" está alto versus quando está baixo. Ele captura esses relacionamentos contextuais automaticamente.

02 / 05

Organizando Informações Financeiras

Antes do SAINT aprender qualquer coisa, precisamos organizar nossos dados. Cada linha representa uma empresa em um ponto no tempo. Cada coluna é uma métrica financeira — o que chamamos de features.

📊
Métricas de Valor
P/L, preço/livro, dividend yield — a ação está cara ou barata?
📈
Métricas de Crescimento
Crescimento de receita, lucros — a empresa está expandindo?
⭐
Métricas de Qualidade
Rentabilidade, retorno sobre equity — a empresa é bem administrada?
🚀
Métricas de Momentum
Momentum de preço, volatilidade — como a ação está se movendo?
Como Features se Tornam "Tokens"

Tokenização

Cada feature é convertida em uma representação numérica — uma lista de 256 números. Isso é chamado de embedding. Pense como traduzir o número bruto "15,2" em uma descrição mais rica que captura seu significado.

O que é um Embedding?
Um embedding é uma forma de representar informações como números. Ao invés de apenas armazenar "P/L = 15,2", armazenamos 256 números que codificam o que "15,2" significa em contexto — é alto ou baixo? Qual faixa é típica? O que ele prediz? O modelo aprende essas representações automaticamente.
03 / 05

Como o SAINT Aprende Relacionamentos

Esta é a parte central da arquitetura SAINT. Depois de converter features em embeddings, o modelo as processa através de 6 camadas de self-attention.

Cada camada faz algo simples mas poderoso: permite que cada feature "faça perguntas" para todas as outras features. "Preço/lucro" pode perguntar para "momentum": "Meu valor significa algo diferente quando ações estão subindo rápido?"

1

Embeddings de Features Entram

Cada feature (P/L, momentum, etc.) agora é um vetor de 256 dimensões

2

Camada de Self-Attention

Cada feature computa seu relacionamento com todas as outras features

3

Processamento Feed-Forward

A representação de cada feature é refinada baseado no que aprendeu

4

Repetir 6 Vezes

Passar por múltiplas camadas permite aprender relacionamentos cada vez mais complexos

5

Token CLS Agrega

Um token especial coleta informações de todas as features em um único resumo

Por que Múltiplas Camadas?
Uma camada pode aprender relacionamentos simples: "momentum alto + volatilidade baixa" é bom. Seis camadas mais profundo, o modelo pode aprender combinações complexas: "momentum alto + volatilidade baixa + valorização barata + rentabilidade forte" pode ser ainda melhor, mas só em certas condições de mercado.
Aspecto Abordagem Tradicional Abordagem SAINT
Relacionamentos de features Assume independência ou interações manualmente criadas Aprende interações automaticamente dos dados
Contexto Mesma feature sempre tem o mesmo efeito O significado da feature muda baseado no contexto
Não-linearidade Limitada ou manualmente especificada Rede profunda captura padrões complexos
04 / 05

Como o Modelo Aprende

SAINT usa aprendizado auto-supervisionado — uma forma chique de dizer que ele aprende com os próprios dados, sem rótulos explícitos. Faz isso através de duas tarefas de treinamento.

🔒
Tarefa 1: Reconstrução Masc.
Esconde algumas features e pede ao modelo para adivinhá-las. Isso força a aprender relacionamentos — você não consegue adivinhar um valor escondido se não entende como ele se relaciona com os visíveis.
⏭️
Tarefa 2: Predição Temporal
Mostra os dados deste mês e pede ao modelo para prever os features do próximo mês. Isso ensina sobre dinâmicas — como indicadores evoluem com o tempo.

Ambas tarefas rodam por 10 epochs cada. Um epoch significa passar pelo dataset completo uma vez, com os dados embaralhados aleatoriamente cada vez para ajudar o modelo a generalizar.

Por que Auto-Supervisionado?
Não precisamos rotular manualmente os dados como "esta é uma boa ação" ou "esta é uma má ação". Em vez disso, criamos tarefas de aprendizado dos próprios dados. Ao aprender a reconstruir features mascaradas e prever valores futuros, o modelo constrói representações ricas que acontecem de ser úteis para prever retornos.
05 / 05

Do Modelo à Carteira

Depois do treinamento, usamos o modelo SAINT para gerar previsões. Mas não apenas usamos a saída bruta do modelo — combinamos com outras técnicas em uma abordagem de dois estágios.

1

Extrair Embeddings CLS

Depois do SAINT processar todas as features, o token CLS contém uma representação comprimida de tudo que o modelo aprendeu sobre aquela empresa naquele momento

2

Redução de Dimensionalidade

Usa PCA para reduzir de 256 para 100 dimensões, mantendo as informações mais importantes

3

Ensemble Random Forest

Treina 200 árvores de decisão nos embeddings para prever retornos reais

4

Suavização KNN

Aplica k-nearest neighbors para suavizar previsões, reduzindo ruído

5

Construir Carteira

Converte previsões em posições com controles de risco (64+ posições, long/short balanceado)

Por que Dois Estágios?

SAINT constrói representações ricas de empresas através de aprendizado auto-supervisionado. Mas para a previsão final, usamos Random Forest — é robusto, lida bem com ruído e fornece quantificação natural de incerteza. A suavização KNN adiciona uma camada final de estabilidade. Esta combinação performa melhor que cada abordagem sozinha.

Gestão de Risco
O modelo gera previsões, mas não as seguimos cegamente. Limitamos tamanhos máximos de posição, exposição de setor, e mantemos diversificação. A IA é uma ferramenta que informa decisões — julgamento humano e controles de risco continuam essenciais.
